# Blog 项目前后端 Docker 部署命令版（固定目录结构）

适用目标目录（按你的要求）：
- 后端目录：`/root/project/blog_project`
- 前端目录：`/root/project/blog_vue`
- 前端只上传构建产物：`/root/project/blog_vue/dist`

说明：
- 文档为命令行级别，按顺序复制执行。
- 默认使用 Docker Compose 部署。
- 默认数据库是“外部 MySQL”（可为宿主机上的容器映射端口，如 `10082`）。

---

## 0) 先回答你关心的问题：后端需要上传什么？

前端可以只上传 `dist`，但后端若采用“服务器构建镜像”的方案，需要上传源码最小集合：

- `manage.py`
- `requirements.txt`
- `config/`
- `apps/`
- `templates/`（如有）
- `static/`（如有）
- `.env`（生产环境变量，建议服务器单独维护）

你也可以不上传源码，改为上传后端镜像包（`docker save` 导出的 tar），但那是另一套发布流程。本文档采用更直观的“上传源码并在服务器构建”。

## 1) 变量准备（先改成你自己的值）

```bash
export DOMAIN='your-domain.com'
export EMAIL='you@example.com'

# 外部 MySQL 连接参数（如果 MySQL 在宿主机 Docker 并映射 10082，按默认即可）
export DB_HOST='host.docker.internal'
export DB_PORT='10082'
export DB_NAME='blog_project'
export DB_USER='blog_app'
export DB_PASSWORD='change-me'

# Django
export DJANGO_SECRET_KEY='change-this-to-a-strong-secret-key'
```

## 3) 上传代码/文件


## 4) 生成后端 Dockerfile 与启动脚本

```bash
cat > /root/project/blog_project/docker-entrypoint.sh <<'EOF2'
#!/bin/sh
set -e

python manage.py migrate --noinput
python manage.py collectstatic --noinput

exec gunicorn config.wsgi:application \
  --bind 0.0.0.0:8000 \
  --workers 3 \
  --timeout 120
EOF2
chmod +x /root/project/blog_project/docker-entrypoint.sh
```

```bash
cat > /root/project/blog_project/Dockerfile <<'EOF2'
FROM python:3.12-slim

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential curl \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt /app/requirements.txt
RUN pip install --no-cache-dir -r requirements.txt gunicorn

COPY . /app
RUN chmod +x /app/docker-entrypoint.sh

EXPOSE 8000
CMD ["/app/docker-entrypoint.sh"]
EOF2
```

## 5) 写后端 .env（放在后端目录）

```bash
cat > /root/project/blog_project/.env <<EOF2
DJANGO_SETTINGS_MODULE=config.settings.prod
DJANGO_SECRET_KEY=${DJANGO_SECRET_KEY}
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=${DOMAIN}

DB_HOST=${DB_HOST}
DB_PORT=${DB_PORT}
DB_NAME=${DB_NAME}
DB_USER=${DB_USER}
DB_PASSWORD=${DB_PASSWORD}

CORS_ALLOWED_ORIGINS=
CORS_ALLOWED_ORIGIN_REGEXES=^https?://${DOMAIN//./\\.}$
EOF2
```

## 6) 生成 Nginx 配置（先 HTTP，便于签证书）

```bash
mkdir -p /root/project/deploy_blog/nginx
```

```bash
cat > /root/project/deploy_blog/nginx/default.conf <<'EOF2'
server {
    listen 80;
    server_name _;

    client_max_body_size 20m;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location /static/ {
        alias /srv/static/;
        access_log off;
        expires 7d;
    }

    location /media/ {
        alias /srv/media/;
        access_log off;
        expires 7d;
    }

    location /api/ {
        proxy_pass http://backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
EOF2
```

---

## 7) 生成 docker-compose.yml

```bash
cat > /root/project/deploy_blog/docker-compose.yml <<'EOF2'
services:
  backend:
    build:
      context: /root/project/blog_project
      dockerfile: Dockerfile
    container_name: blog_backend
    restart: always
    env_file:
      - /root/project/blog_project/.env
    extra_hosts:
      - "host.docker.internal:host-gateway"
    expose:
      - "8000"
    volumes:
      - /root/project/blog_project/runtime/media:/app/media
      - /root/project/blog_project/runtime/staticfiles:/app/staticfiles
      - /root/project/blog_project/runtime/logs:/app/logs

  frontend:
    image: nginx:1.27-alpine
    container_name: blog_frontend
    restart: always
    depends_on:
      - backend
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - /root/project/blog_vue/dist:/usr/share/nginx/html:ro
      - /root/project/deploy_blog/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
      - /root/project/blog_project/runtime/media:/srv/media:ro
      - /root/project/blog_project/runtime/staticfiles:/srv/static:ro
      - certbot_etc:/etc/letsencrypt
      - certbot_www:/var/www/certbot

  certbot:
    image: certbot/certbot
    volumes:
      - certbot_etc:/etc/letsencrypt
      - certbot_www:/var/www/certbot

volumes:
  certbot_etc:
  certbot_www:
EOF2
```

## 9) 申请 HTTPS 证书

确保域名已解析到服务器公网 IP 后执行：

```bash
cd /root/project/deploy_blog
docker compose run --rm certbot certonly \
  --webroot -w /var/www/certbot \
  -d "$DOMAIN" \
  --email "$EMAIL" \
  --agree-tos --no-eff-email
```

## 10) 切换为 HTTPS Nginx 配置

```bash
cat > /root/project/deploy_blog/nginx/default.conf <<EOF2
server {
    listen 80;
    server_name ${DOMAIN};

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://\$host\$request_uri;
    }
}

server {
    listen 443 ssl http2;
    server_name ${DOMAIN};

    ssl_certificate /etc/letsencrypt/live/${DOMAIN}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/${DOMAIN}/privkey.pem;

    client_max_body_size 20m;

    location /static/ {
        alias /srv/static/;
        access_log off;
        expires 7d;
    }

    location /media/ {
        alias /srv/media/;
        access_log off;
        expires 7d;
    }

    location /api/ {
        proxy_pass http://backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }

    location / {
        root /usr/share/nginx/html;
        try_files \$uri \$uri/ /index.html;
    }
}
EOF2
```

```bash
cd /root/project/deploy_blog
docker compose up -d frontend
docker compose exec frontend nginx -t
docker compose exec frontend nginx -s reload
```
